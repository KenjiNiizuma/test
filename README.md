# 1 概要

SCCの状況照会画面では、クライアントからTWX-21 / SCCGWへのアップロードおよびダウンロード状況を確認する機能を提供します。

# 2 前提条件および制約条件

## 2.2 入力禁止文字について

検索条件には、HTMLのタグに相当する文字列(\/:;?"<>)を持つデータを使用することができません。入力テキストには入力しないでください。
使用した場合、画面への表示が正しく行われない可能性があります。

## 2.3 その他の前提条件および制約条件

(１)画面に表示される文字コードおよび入力できる文字コードはUTF-8です。また、言語は日本語・中国語・英語の3言語です。
状況照会画面操作時は、最初にログインする際に日本語／中国語／英語のどの言語のログイン画面を使用するかを選択することで、使用言語が決まります。

(2)本機能の使用者は、TWX-21 / GW状況照会の利用権限を持つ利用者、およびTWX運用者です。
TWX運用管理者は、利用者の使用可能機能に加え、送受信データの内容を取得することができます。
TWX運用者権限を持つユーザIDは複数設定でき、設定ファイルで指定します。

## 2.4 DBアクセス

接続先DBの情報は設定ファイルで管理する。
Direct / Connectを利用して、AWS側で稼働するPostgreSQL(Amazon Aurora)と通信する。
SQLにはオプションを付与せず、デフォルト（排他制御なし）で実行する。
ダーティリードは実施しない。

### 5.2.2 ログイン認証について

サーバでのログイン認証は、portalで行う。
PortalのリンクによってリアーキGW状況照会画面に遷移した際、セッション情報からログインID、
パスワード、言語設定を取得する。
取得後、MARSDB(Postgres)にアクセスし、各種情報を取得する。

(1) 会員IDの取得
SELECT / MEMBER_ID / FROM / MARS_USER_INFO / WHERE / LOGIN_ID = [セッションから取得したログインID] AND / PASSWORD = [セッションから取得したパスワードをMD5ハッシュ化した値]

取得結果が0件の場合は、ユーザ未登録エラー（認証エラー）とする。
SSO連携ログインの場合は、条件句からPASSWORDを除外する。

(2) 企業名・事業所名の取得
MARS_MEMBER_INFOテーブルにアクセスし、COMPANY_NAME（企業名）とOFFICE_NAME
（事業所名）を取得する。
SELECT / COMPANY_NAME, OFFICE_NAME / FROM / MARS_MEMBER_INFO / WHERE / MEMBER_ID = [(1)で取得した会員ID]

### 5.6.2 ファイルダウンロードについて

TWX運用者の場合、本画面上のファイル名をクリックすると、アップロードファイルをダウンロー
ドできます。ただし、添付ファイルはダウンロードできません。
ファイル名をクリックした際の処理の流れは下記のとおり。

（１）ダウンロード対象ファイルのサイズ＞ダウンロードサイズ上限値 / の場合、ダウンロードエラー
とする。（メッセージID：####120005-I）（ファイルサイズは検索結果一覧画面表示時にDBから取
得しておく。）
（２）サーバにダウンロードのリクエスト情報を送信。
（３）サーバはDBからS3のオブジェクトキーを取得し、S3からファイルをストリーミング取得。

|   |   |
| --- | --- |
| Select / FILE_ACTUAL_OBJECT_KEY / From / UPLOAD_FILE_SPLIT_DETAIL / Where / DATA_ID / = ‘送信履歴検索で取得したセンタ内受付ID(UPLOAD_STATUS. DATA_ID)’ / AND / UPLOAD_FILE_SPLIT_DETAIL.DATA_SPLIT_ID / = 0 | ←要確認後 |

|   |   |
| --- | --- |
| ファイル名は「S3に格納されているzipファイル名（拡張子除外）.zip」。 | ←吉田さんに要確認 |

すでにzip圧縮されているファイルの前提。
メモリに取得した後、HTTPレスポンスとして返却。（サーバ側のワークには保存しない）
（４）（３）のファイルのダウンロードを実行し、クライアントの保存先選択ダイアログで指定された
指定された保存先に格納
※保存先選択ダイアログのファイル名には、検索結果一覧画面に表示したファイル名.zipを設定する。

### 5.6.3 状態更新について

TWX運用者の場合、アップロードファイルの状態を変更できます。
状態変更の処理の流れは下記のとおり。

（１）クライアント側で、更新チェックボックスが１個以上チェックされていることを確認。１個もチ
ェックされていない場合、エラーダイアログを表示（メッセージID：####110012-E）
（２）サーバに状態更新のリクエスト情報（送信受付ID、状態の変更値など）を送信
なお、状態変更されるのは当該ページの案件のみ。つまり、複数ページまたがりの状態変更はでき
ない。
（３）サーバにて状態更新を実行
現在の状態の種類に関わらず、状態変更を行える。
状態更新が同時実行された場合は、先に実行された状態変更のupdate文がcommitされるまで、
次のupdateは待つ。
また、状態更新実行時に、同時にサーバ内振分け処理が実行されているかどうかのチェック処理も
行わない。本状態更新機能使用前に、サーバ内振分け処理が実行されていないことを、TWX運用者
が確認すること。（本状態更新機能とサーバ内振分け処理が同時実行された場合、DB内のデータに
不整合が発生する可能性がある）

### 5.8.2 ファイルダウンロードについて

TWX運用者の場合、本画面上のファイル名をクリックすると、アップロードファイルをダウンロー
ドできます。ただし、添付ファイルはダウンロードできません。
ファイル名をクリックした際の処理の流れは下記のとおり。

（１）ダウンロード対象ファイルのサイズ＞ダウンロードサイズ上限値 / の場合、ダウンロードエラー
とする。（メッセージID：####120005-I）（ファイルサイズは検索結果一覧画面表示時にDBから取
得しておく。）
（２）サーバにダウンロードのリクエスト情報を送信。
（３）サーバはDBからS3のオブジェクトキーを取得し、S3からファイルをストリーミング取得。

Select / FILE_DATA_OBJECT_KEY / From / DATA_FILE_OBJECT_KEY / Where / CENTER_ID = ‘案件検索で取得したセンタ内ID(FILE_KANRI_UL.CENTER_ID)’

ファイル名は「S3に格納されているzipファイル名（拡張子除外）.zip」。
すでにzip圧縮されているファイルの前提。
メモリに取得した後、HTTPレスポンスとして返却。（サーバ側のワークには保存しない）
（４）（３）のファイルのダウンロードを実行し、クライアントの保存先選択ダイアログで指定された
指定された保存先に格納
※保存先選択ダイアログのファイル名には、検索結果一覧画面に表示したファイル名.zipを設定する。

### 5.10.3 状態更新について

TWX運用者の場合、アップロードファイルの状態を変更できます。
状態変更の処理の流れは下記のとおり。

（１）クライアント側で、更新チェックボックスが１個以上チェックされていることを確認。１個もチ
ェックされていない場合、エラーダイアログを表示（メッセージID：####110012-E）
（２）サーバに状態更新のリクエスト情報（送信受付ID、状態の変更値など）を送信
なお、状態変更されるのは当該ページの案件のみ。つまり、複数ページまたがりの状態変更はでき
ない。
（３）サーバにて状態更新を実行
現在の状態の種類に関わらず、状態変更を行える。
状態更新が同時実行された場合は、先に実行された状態変更のupdate文がcommitされるまで、
次のupdateは待つ。
また、状態更新実行時に、同時にサーバ内振分け処理が実行されているかどうかのチェック処理も
行わない。本状態更新機能使用前に、サーバ内振分け処理が実行されていないことを、TWX運用者
が確認すること。（本状態更新機能とサーバ内振分け処理が同時実行された場合、DB内のデータに
不整合が発生する可能性がある）
